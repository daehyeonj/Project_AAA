# Batch 71 Beta Signature Demo Playbook

## Scope

- Signature pair: `city-b -> dungeon-beta`
- Recommended first route to present: `Beta safe` (`Guarded Path`)
- Alternate contrast route: `Beta risky` (`Greedy Path`)
- Promise to prove:
  - `City B pressure -> guarded vs greedy route answer -> burst-window combat payoff -> return evidence -> next answer`

## Why Guarded First

- `Guarded Path` is the cleanest proof that route choice is an operating answer, not only a bigger-number route.
- It lets the presenter say "City B needs restart pressure answered without stacking more strain" before showing that the route preview, party fit, and follow-up text all reinforce the same story.
- Keep `Greedy Path` as the fast contrast when the audience asks why the run should not always chase the highest payout.

## Five-Step Demo Flow

### 1. World Board

- Screen goal:
  - show that City B is readable as the current pressure source
  - show that the board already points at a route answer instead of only showing passive status
- Read on screen:
  - `Pressure Board`
  - `Why City Matters`
  - `Route Answer`
  - `Party Readiness`
  - `Latest Evidence`
  - `Next Action`
- Presenter line:
  - "City B is the priority because the refined mana line needs a restart, and the board is already turning the last return into the next answer."
- Proof on screen:
  - the world summary ties urgency, latest evidence, and answer together instead of treating the last run as a dead-end log

### 2. Expedition Prep

- Screen goal:
  - prove that party differentiation and route-fit are already on the same decision surface
- Read on screen:
  - `Recommendation`
  - `Reason`
  - `Expected Need Impact`
  - `Projected Preview`
  - `Route Fit`
- Presenter line:
  - "This is not just 'pick safe or risky'; the prep board tells us why the current party fits this answer and what kind of follow-up pressure it will leave behind."
- Proof on screen:
  - recommendation text, expected need impact, and route-fit text are all describing the same City B restart problem

### 3. Route Answer

- Screen goal:
  - prove that `Guarded` versus `Greedy` is an authored scenario split, not a raw reward-table swap
- Read on screen:
  - `Route Preview`
  - `Reward Preview`
  - `Event Preview`
- Presenter line:
  - "Guarded is the steady restart line; Greedy is the raid line that restarts faster but pushes more recovery debt into the next answer."
- Proof on screen:
  - route preview gives the scenario identity
  - reward preview shows the payout shape
  - event preview shows the combat/event posture the route is asking the player to accept

### 4. Battle Payoff

- Screen goal:
  - prove that combat payoff is readable as `read intent -> create window -> cash payoff`
- Read on screen:
  - command/context effect text for `Power Strike`, `Weak Point`, `Arcane Burst`, and `Radiant Hymn`
  - exposed / payoff-ready readback on the current target
- Presenter line:
  - "The battle pleasure is not hidden math. We read intent, open the burst window, and then cash it in with the right follow-through."
- Proof on screen:
  - `Power Strike` opens the window
  - `Weak Point` cashes it in
  - `Arcane Burst` rewards exposed targets
  - `Radiant Hymn` stabilizes the window instead of breaking the loop

### 5. Result And Return

- Screen goal:
  - prove that the run closes back into world pressure instead of ending as an isolated reward screen
- Read on screen:
  - `Latest Return`
  - `Outcome Readback`
  - `Corrective Follow-Up`
  - `Return Board Hint`
  - then back on the world board: `Pressure Board`, `Latest Evidence`, `Next Action`
- Presenter line:
  - "The run comes back as evidence. The city absorbs the return, the board restates what changed, and it already points toward the next answer."
- Proof on screen:
  - result screen carries latest return + follow-up
  - world board reuses that evidence as the next pressure/answer context

## Recommended Spoken Arc

1. `World`: "City B needs a restart, and the board is already telling us what kind of answer it wants."
2. `Prep`: "This party is not generic; the recommendation and route-fit are reacting to the actual crew."
3. `Route`: "Guarded versus Greedy is a scenario choice about restart posture, not just a loot slider."
4. `Battle`: "Burst windows make the chosen posture readable in play, not only in text."
5. `Result`: "The return becomes world evidence, so the loop closes into the next decision."

## Tight-Time Fallback

- If time is short, show this compressed path:
  1. City B world board with `Pressure Board`, `Latest Evidence`, `Next Action`
  2. Prep board with `Recommendation`, `Expected Need Impact`, `Route Fit`
  3. One battle moment with `Power Strike -> Weak Point`
  4. Result screen `Latest Return` + `Corrective Follow-Up`
- Tight-time presenter line:
  - "This loop is: pressure creates an answer, the answer changes the run posture, battle makes that posture felt, and the result comes back as the next piece of evidence."

## When To Pivot To Greedy

- Use `Beta risky` if the audience already accepts the world/prep logic and wants the sharper contrast route.
- In that version, emphasize:
  - faster restart
  - rougher follow-up
  - stronger short-window combat commitment
