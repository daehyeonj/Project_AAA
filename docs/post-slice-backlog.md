# Project AAA Post-Slice Backlog Split

## If Manual Evaluation Fails Before Sign-Off

These are still slice-scope follow-ups, but only if manual evaluation says the current vertical slice is not yet readable enough:

- tighten one golden-path clarity blocker at a time
- improve one missing mission/result anchor if the player cannot explain the loop after a full run
- close one stale selection or refresh issue if the re-entry path feels misleading in manual play

## Post-Slice Expansion Candidates

These should wait until after the vertical slice exit decision:

- content expansion: more cities, dungeons, routes, encounters, events
- UX polish: cleaner HUD, less debug-heavy text, better presentation hierarchy
- optional wording polish beyond the now-guarded prep re-entry why-now summary that batch 21 compressed to fit the prep prompt
- system specialization: richer city policy, deeper expedition planning, broader battle roles
- save/load and session persistence
- broader localization polish and content coverage
- balancing, pacing, and economy tuning beyond the current sample loop
- telemetry, analytics, and broader validation automation

## Why This Split Exists

- Batch 20 is for exit review and manual-evaluation readiness.
- Anything that does not directly affect slice readability or golden-path continuity should be treated as post-slice work.
- Direction changes after batch 20 should be informed by manual play feedback, not assumed in advance.
